## 1. 项目愿景与目标

本项目旨在为 iBizSim 企业竞争模拟竞赛打造一套**多决策域、三模式、双角色**的智能决策辅助系统。系统将覆盖竞赛全流程，通过数据分析、智能推荐和AI辅助，帮助**学员**快速制定高质量决策，同时赋能**导师**高效设计和分发决策策略，最终形成一个可持续的策略生态系统。

## 2. 系统架构

### 2.1. 技术栈选型

系统将采用**混合模式（Hybrid Model）**架构，兼顾低成本快速迭代和强大的可扩展性。

- **前端 (Frontend)**: `Vite + React + TypeScript`
  - **职责**: 承载所有核心计算逻辑（约束验证、公式求解）、UI交互、数据可视化。
  - **优势**: 部署于 GitHub Pages 或类似静态托管平台，零服务器成本，计算响应快。

- **后端 (Backend)**: `Node.js / Python` (待定)
  - **职责**: 仅承载必须由服务端处理的业务，包括：
    1.  **AI大模型调用**: 封装 OpenAI 等 API Key，避免在前端暴露。
    2.  **数据持久化**: 存储用户账号、方案模板、付费状态等核心数据。
    3.  **权限与付费控制**: 管理方案密钥、下载次数、积分等。
  - **优势**: 功能强大，数据安全，支持复杂的商业逻辑。

### 2.2. 核心数据模型：三层架构

系统核心是解耦的**三层数据模型**，确保了配置与计算的分离。

```mermaid

    A[1. EnterpriseState<br>(企业当前状态)] --> B[2. DecisionPlan<br>(用户决策方案)];
    B --> C[3. IBizSimDecision<br>(最终决策数据)];

    subgraph "用户输入 / 报表导入"
        A
    end

    subgraph "用户配置 / 导师模板"
        B
    end

    subgraph "提交给竞赛系统"
        C
    end

    style A fill:#f9f,stroke:#333,stroke-width:2px
    style B fill:#ccf,stroke:#333,stroke-width:2px
    style C fill:#9f9,stroke:#333,stroke-width:2px
```

| 层级 | 名称 | 说明 | 谁填写 |
|:---|:---|:---|:---|
| **1. 状态层** | `EnterpriseState` | 企业当前状态，包含从竞赛报表中获取的所有初始数据。 | 学员手动抄入或通过“一键导入报表”功能自动填充。 |
| **2. 方案层** | `DecisionPlan` | 用户的决策意图和策略配置，描述**如何得出**决策值。 | 学员手动配置，或从预设/导师的策略模板库中选择。 |
| **3. 决策层** | `IBizSimDecision` | 最终生成的、可直接提交给竞赛系统的完整决策数据。 | 由前端计算引擎根据 `State` 和 `Plan` 自动计算生成。 |

## 3. 数据结构定义 (v1.0)

以下是覆盖所有决策域的完整数据结构定义，将作为系统统一的数据契约。

*（为保持文档简洁，具体 TypeScript 接口定义见附录或代码库中的 `types.ts` 文件，此处仅作结构示意）*

- **`EnterpriseState.ts`**: 包含 `lastPeriodFinancials`, `marketShare`, `productInventory` 等从报表获取的数据结构。
- **`DecisionPlan.ts`**: 包含 `productionPlan`, `marketingPlan` 等，其叶子节点为 `CellBehavior` (`{mode: 'manual' | 'solve' | 'ai_solve', ...}`).
- **`IBizSimDecision.ts`**: 包含 `production`, `marketing` 等，其叶子节点为最终的 `number` 类型决策值。

## 4. 用户角色与运营方案

系统将服务于两种核心用户，并通过“方案市场+积分制”的 Freemium 模型实现商业闭环。

### 4.1. 用户画像与需求

| 用户角色 | 核心需求 | 对应功能 |
|:---|:---|:---|
| **学员** | 快捷、傻瓜式操作，快速获得高质量决策建议。 | 手动模拟器、一键使用内置/导师的推荐方案。 |
| **导师/高级用户** | 高效、灵活地设计、测试、验证和分发自己的决策策略。 | `DecisionPlan` 可视化编辑器、方案发布与管理后台。 |

### 4.2. 商业模式：Freemium + 方案市场

| 功能层级 | 免费用户 | 付费用户（订阅制） | 方案作者收益 |
|:---|:---|:---|:---|
| 手动模拟器 | 无限使用 | 无限使用 | - |
| 公式推荐（内置策略） | 每期3次免费推荐 | 无限推荐 | - |
| AI辅助模式 | 不可用 | 无限使用 | - |
| 公开方案下载 | 每月5个免费额度 | 无限下载 | **作者获得积分/分成** |
| 私有方案（密钥分享） | 密钥持有者免费使用 | 同左 | 作者自行决定是否收费 |
| 方案发布权限 | 最多发布2个方案 | 无限发布 | - |

## 5. 开发策略与实施路线图

我们将采用**“完整结构 + 渐进式填充”**的策略进行开发。

### 5.1. 开发策略

1.  **架构先行**: 优先搭建好完整的前后端项目骨架和三层数据结构。
2.  **模块化实现**: 逐个决策域实现其完整闭环（UI → 计算 → 反馈）。
3.  **参考但不依赖**: 第三方 Excel 公式仅作为设计参考，核心求解逻辑（如约束方程最优解）由我们自行研发，确保健壮性和准确性。

### 5.2. 实施路线图 (Roadmap)

| 阶段 | 核心任务 | 交付成果 | 预计角色/技术 |
|:---|:---|:---|:---|
| **Phase 1: 基础框架搭建** | 1. 初始化前后端项目<br>2. 定义三层数据结构 TS 类型<br>3. 实现用户认证基础 | - Git 仓库<br>- `types.ts`<br>- 登录/注册页面 | 全栈 | 
| **Phase 2: 生产决策闭环** | 1. 实现生产决策的 UI<br>2. 开发生产决策求解器<br>3. 实现 `DecisionPlan` 编辑器（生产部分） | 可独立运行的生产决策模拟器 | 前端、算法 | 
| **Phase 3: 核心功能完善** | 1. 逐个实现其他决策域<br>2. 对接 AI 辅助模式 API<br>3. 实现报表数据导入功能 | 功能完整的单用户版决策系统 | 前端、后端、算法 | 
| **Phase 4: 运营与生态** | 1. 开发方案市场<br>2. 实现付费与积分系统<br>3. 建设导师管理后台 | 支持方案分享和付费的完整生态 | 全栈、产品 |
