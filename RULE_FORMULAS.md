## 2. 产品规格参数（全局常量）

以下参数在所有9期中保持不变，是计算产量消耗资源的基础。每个工作时间单位为 **520 小时**。

| 产品 | 机器需求 (时/单位) | 人力需求 (时/单位) | 原材料 (单位) | 机器系数 | 人力系数 |
|:---:|:---:|:---:|:---:|:---:|:---:|
| **A** | 110 | 80 | 500 | `110/520 = 0.2115` | `80/520 = 0.1538` |
| **B** | 150 | 100 | 800 | `150/520 = 0.2885` | `100/520 = 0.1923` |
| **C** | 180 | 110 | 1600 | `180/520 = 0.3462` | `110/520 = 0.2115` |
| **D** | 280 | 140 | 2500 | `280/520 = 0.5385` | `140/520 = 0.2692` |

> **机器系数** = 生产一个产品需要的机器数 = 机器需求 / 520
> **人力系数** = 生产一个产品需要的工人数 = 人力需求 / 520

## 3. 右侧参数区公式（每期资源面板）

每期的右侧面板包含7个关键参数，控制该期的资源供给。以下公式适用于所有9期。

### 3.1 通用公式

| 参数名 | 公式 | 颜色 | 说明 |
|:---|:---|:---:|:---|
| **本期机器** | P1: `Input` / P2+: `P(n-1).本期机器 + P(n-2).本期购买` | P1:黄 / P2+:蓝 | P1由用户输入；P2起为公式（购买机器两期后到货） |
| **本期购买** | `Input` | 橙 | 用户决定本期购买的机器数量 |
| **期初人数** | P1: `Input` / P2+: `P(n-1).期初人数 + P(n-1).本期雇佣 - P(n-1).本期解雇` | P1:黄 / P2+:蓝 | P1由用户输入；P2起为上期结转 |
| **最少解雇** | `⌈期初人数 × 3%⌉` | 蓝 | 向上取整，每期必须至少解雇3%的员工 |
| **本期解雇** | `Input`（默认 = 最少解雇） | 橙 | 用户决定，不得低于最少解雇 |
| **最大雇佣** | `⌊期初人数 × 50%⌋` | 蓝 | 向下取整，每期最多雇佣50%的新员工 |
| **本期雇佣** | `Input` | 橙 | 用户决定，不得超过最大雇佣 |
| **总可用人数** | `期初人数 - 本期解雇 + 本期雇佣 × 0.25` | 蓝 | 新雇佣工人效率仅为25% |

## 5. 约束检查公式（6个关键检查点）

以下6个约束检查点在 **所有9期中公式完全一致**，是判断排产方案是否可行的核心依据。约束值 **必须 >= 0**，否则表示资源超限。

### 5.1 约束公式定义

为简化表达，定义以下中间变量：

| 中间变量 | 定义 |
|:---|:---|
| `一正消耗人力` | `Σ(一班各产品产量 × 对应人力系数)` |
| `二正消耗人力` | `Σ(二班各产品产量 × 对应人力系数)` |
| `一加消耗人力` | `Σ(一加各产品产量 × 对应人力系数)` |
| `二加消耗人力` | `Σ(二加各产品产量 × 对应人力系数)` |
| `一正消耗机器` | `Σ(一班各产品产量 × 对应机器系数)` |
| `二正消耗机器` | `Σ(二班各产品产量 × 对应机器系数)` |
| `一加消耗机器` | `Σ(一加各产品产量 × 对应机器系数)` |
| `二加消耗机器` | `Σ(二加各产品产量 × 对应机器系数)` |

### 5.2 六个约束检查点

| 序号 | 约束名称 | Excel 位置 | 计算公式 | 含义 |
|:---:|:---|:---:|:---|:---|
| **C1** | 可用人数-一班后 | 可用人数行-第一班列 | `总可用人数 - 一正消耗人力 - 二正消耗人力` | 一班和二班共享人力池，一班结束后剩余人力 |
| **C2** | 可用人数-一加后 | 可用人数行-一加列 | `一正消耗人力 - 一加消耗人力 × 2` | 一加只能用一班工人，且加班消耗翻倍 |
| **C3** | 可用人数-二班后 | 可用人数行-二班列 | **禁区（暗色，不计算）** | Excel 中标记为暗色(51,51,51) |
| **C4** | 可用人数-二加后 | 可用人数行-二加列 | `二正消耗人力 - 二加消耗人力 × 2` | 二加只能用二班工人，且加班消耗翻倍 |
| **C5** | 可用机器-一班后 | 可用机器行-第一班列 | `本期机器 - 一正消耗机器` | 一班结束后剩余机器 |
| **C6** | 可用机器-一加后 | 可用机器行-一加列 | **禁区（暗色，不计算）** | Excel 中标记为暗色(51,51,51) |
| **C7** | 可用机器-二班后 | 可用机器行-二班列 | `本期机器 - 二正消耗机器 - 一加消耗机器 × 2` | 一加的机器消耗翻倍后从二班机器池中扣除 |
| **C8** | 可用机器-二加后 | 可用机器行-二加列 | `本期机器 - 二加消耗机器 × 2` | 二加的机器消耗翻倍 |

> **关键理解**: 加班班次（一加、二加）的资源消耗系数为正班的 **2倍**，这是因为加班时间仅为正班的一半，但需要相同的机器和人力在更短时间内完成生产。

## 9. 约束公式展开示例

以某期为例，假设排产数据如下：

| 班次 | A | B | C | D |
|:---:|:---:|:---:|:---:|:---:|
| 一班 | a1 | b1 | c1 | d1 |
| 一加 | a2 | b2 | c2 | d2 |
| 二班 | a3 | b3 | c3 | d3 |
| 二加 | a4 | b4 | c4 | d4 |

则6个约束的展开公式为：

**C1: 可用人数-一班后**
```
= 总可用人数
  - (a1×0.1538 + b1×0.1923 + c1×0.2115 + d1×0.2692)
  - (a3×0.1538 + b3×0.1923 + c3×0.2115 + d3×0.2692)
```

**C2: 可用人数-一加后**
```
= (a1×0.1538 + b1×0.1923 + c1×0.2115 + d1×0.2692)
  - (a2×0.1538 + b2×0.1923 + c2×0.2115 + d2×0.2692) × 2
```

**C4: 可用人数-二加后**
```
= (a3×0.1538 + b3×0.1923 + c3×0.2115 + d3×0.2692)
  - (a4×0.1538 + b4×0.1923 + c4×0.2115 + d4×0.2692) × 2
```

**C5: 可用机器-一班后**
```
= 本期机器
  - (a1×0.2115 + b1×0.2885 + c1×0.3462 + d1×0.5385)
```

**C7: 可用机器-二班后**
```
= 本期机器
  - (a3×0.2115 + b3×0.2885 + c3×0.3462 + d3×0.5385)
  - (a2×0.2115 + b2×0.2885 + c2×0.3462 + d2×0.5385) × 2
```

**C8: 可用机器-二加后**
```
= 本期机器
  - (a4×0.2115 + b4×0.2885 + c4×0.3462 + d4×0.5385) × 2
```

## 10. 总结

`rule.xls` 连续排产的核心逻辑可以归纳为以下三点：

**资源迭代**: 上一期的雇佣、解雇、购买决策会影响下一期的期初资源（人力和机器）。期初人数通过 `上期期初 + 上期雇佣 - 上期解雇` 迭代，本期机器通过 `上期机器 + 两期前购买` 迭代。新雇佣工人的效率系数仅为 **0.25**，这是人力规划的关键约束。

**固定约束**: 6个核心的人力/机器约束检查点在所有期数中保持不变。加班班次（一加、二加）的资源消耗为正班的 **2倍**。其中"可用人数-二班后"和"可用机器-一加后"为禁区，不参与计算。

**动态排产**: P1-P5 的排产有明确的颜色指引（黄色为必填，金色为选填），每期的排产模式各不相同。P6-P9 则完全开放，所有产量格均无颜色标记，需要依赖优化算法或手动决策来填补。
