# iBizSim 排产求解器算法详解

> **文档版本**: 1.1
> **更新时间**: 2026-03-05

---

## 1. 概述

本排产求解器旨在为 iBizSim 企业竞争模拟大赛提供一个高效、精确的连续排产方案。其核心目标是在满足所有6个关键约束（3个可用人数约束和3个可用机器约束）的前提下，最大化产能利用率。该算法在 `client/src/lib/engine.ts` 文件中的 `optimizeShiftPlan` 函数及其辅助函数中实现。

算法根据 `rule.xls` 中定义的颜色标记，采用两种不同的策略：

1.  **颜色感知求解算法 (Color-Aware Solver)**：用于处理规则明确的期数（P1-P5），严格遵循 Excel 中定义的单元格颜色（黄色/必填）进行排产。
2.  **通用启发式算法 (Generic Heuristic)**：用于处理无明确颜色规则的期数（P6-P9），采用一套固定的、经过验证的排产模式。

## 2. 核心原则：全局约束感知

与旧版独立计算每个班次产量的算法不同，新版求解器（v0.4.0+）的核心是 **全局约束感知**。这意味着在求解过程中的每一步，算法都会实时计算并验证全部6个约束条件，确保任何中间步骤都不会导致最终方案的不可行。这一原则通过以下机制实现：

- **严格向下取整 (Math.floor)**：在所有产量计算中，均使用 `Math.floor` 代替 `Math.round`，从根本上杜绝了因四舍五入导致的资源超额分配。
- **迭代式约束修正 (`_enforceConstraints`)**：在生成初步方案后，会有一个最终的安全检查环节，通过迭代循环逐步减少产量，直到所有约束值均 `>= 0`。
- **超限惩罚评估 (`evaluatePlan` / `scorePlan`)**：在搜索最优方案时，任何导致约束值为负的方案都会被赋予极高的惩罚分，从而在评估中被自动淘汰。

## 3. 颜色感知求解算法 (P1-P5)

此算法是求解器的精髓所在，专门为规则明确的 P1-P5 设计。

### 3.1. 算法流程：多比例搜索策略

为了寻找全局最优解而非局部最优解，算法采用了一种 **多比例搜索策略** 来寻找全局最优解。

1.  **定义搜索空间**：算法首先定义了一个 `shift1`（第一班）的机器资源使用比例数组 `fractions`（从30%到100%，共15个档位）。

2.  **遍历搜索**：对 `fractions` 中的每一个比例 `frac`，执行以下序贯班次求解流程：

    a.  **求解第一班 (shift1)**：
        - **目标**：充分利用分配到的机器资源（`machines * frac`）。
        - **方法**：调用 `_solveShiftGroupSafe` 函数，为 `shift1` 的所有黄色格子分配产量。

    b.  **求解第二班 (shift2)**：
        - **目标**：在满足机器约束的前提下，尽量用完“一班后可用人数”（`totalAvailableWorkers - shift1LaborUsed`）。
        - **方法**：调用 `_solveShiftGroupDual` 函数，这是一个 **双约束求解器**，它会同时考虑剩余的人力资源和总机器资源，为 `shift2` 的黄色格子分配产量。

    c.  **求解一加班 (ot1)**：
        - **目标**：同时满足“一加后可用人数”和“二班后可用机器”两个约束。
        - **方法**：分别计算这两个约束允许的最大资源量，取其较小值作为 `ot1` 的求解资源上限，然后调用 `_solveShiftGroupSafe` 进行分配。

    d.  **求解二加班 (ot2)**：
        - **目标**：同时满足“二加后可用人数”和“二加后可用机器”两个约束。
        - **方法**：调用 `_solveOt2MultiSafe` 函数，这是一个专门为 `ot2` 设计的多产品、双约束求解器。

    e.  **最终修正**：调用 `_enforceConstraints` 函数对当前生成的方案进行最终验证和微调，确保100%无超限。

    f.  **评估方案**：使用 `scorePlan` 函数计算当前方案的综合评分（所有6个约束残差的绝对值之和）。

3.  **选择最优方案**：在遍历完所有15个比例后，选择综合评分最低（即资源浪费最少）的那个方案作为最终推荐结果。

### 3.2. 关键辅助函数

| 函数名 | 用途 |
|---|---|
| `evaluatePlan` / `scorePlan` | 计算一个排产方案的综合评分。评分基于6个约束的残差绝对值之和，对超限方案施加巨大惩罚，确保优先选择合规方案。 |
| `_solveShiftGroupSafe` | **安全的单资源求解器**。用于在一个资源（如机器或人力）的限制下，为一个班次内的多个产品分配产量。采用多种策略（精确解、遍历搜索、贪心分配）并严格使用 `Math.floor` 来防止超限。 |
| `_solveShiftGroupDual` | **安全的双资源求解器**。在同时考虑人力和机器两种资源限制的情况下，为一个班次分配产量。 |
| `_solveOt2MultiSafe` | **为“二加班”定制的多产品求解器**。专门优化二加班涉及的“二加后可用人数”和“二加后可用机器”两个耦合的约束。 |
| `_enforceConstraints` | **最终约束强制修正函数**。作为安全网，通过迭代减产的方式，修正任何由于浮点精度等问题导致的微小超限，确保最终方案的100%合规性。 |

## 4. 通用启发式算法 (P6-P9)

对于 `rule.xls` 中没有颜色标记的期数，算法回退到一套固定的启发式排产模式。这套模式是基于大量测试和经验总结得出的可靠方案。

- **第一班 (shift1)**：固定生产 C 和 D 产品，通过遍历搜索找到使“一班后可用机器”最接近0的组合。
- **第二班 (shift2)**：固定生产 B 产品，目标是使“一班后可用人数”最接近0。
- **一加班 (ot1)**：固定生产 A 产品，目标是使“二班后可用机器”最接近0。
- **二加班 (ot2)**：固定生产 D 和 B 产品，通过遍历搜索找到一个综合评分（人力残差*3 + 机器残差）最优的组合。

在生成方案后，同样会调用 `_enforceConstraints` 进行最终的约束验证和修正。

## 5. 总结

该求解器通过结合颜色感知的精确求解和通用的启发式策略，实现了对 iBizSim 所有9期排产的全面覆盖。其全局约束感知和多方案搜索的核心设计，确保了在任何参数条件下都能生成一个合规且高效的生产计划。
