# iBizSim 连续排产规则计算器 - 设计方案

## 项目背景
这是一个企业竞争模拟大赛的连续排产规则计算工具，需要展示9期排产数据、多班次产能分析、排班成本对比等内容。目标用户是参赛团队中负责生产的队员。

## 方案一：工业数据仪表盘（Industrial Data Dashboard）

**设计运动**: 新工业主义（Neo-Industrial），融合工厂控制室的功能美学与现代数据可视化

**核心原则**:
1. 信息密度优先 - 在有限空间内展示最大量的有用数据
2. 功能驱动美学 - 每个视觉元素都服务于数据理解
3. 层级清晰 - 通过色彩深浅和尺寸建立信息优先级
4. 实时感 - 界面给人"活的"感觉，数据随交互动态变化

**色彩哲学**: 深色背景（#0F172A 深海蓝黑）搭配高对比度的数据色彩。主色调使用琥珀色（#F59E0B）代表"生产/能量"，辅以青色（#06B6D4）代表"资源/效率"，红色（#EF4444）代表"警告/瓶颈"。背景使用微妙的蓝灰渐变营造深度。

**布局范式**: 左侧固定导航栏 + 右侧多面板网格布局。使用可折叠的卡片系统，用户可以自由展开/收起不同数据模块。顶部有期数选择器作为全局控制。

**签名元素**:
1. 仿LED数字显示的关键指标卡片
2. 微型火花线（sparkline）嵌入数据表格中

**交互哲学**: 悬停即预览，点击即深入。数据卡片悬停时显示详细工具提示，点击展开完整分析面板。

**动画**: 数字变化时使用滚动计数器效果，图表数据使用从左到右的渐入动画，面板切换使用平滑的滑动过渡。

**字体系统**: 标题使用 JetBrains Mono（等宽字体，工业感），正文使用 Noto Sans SC，数据数字使用 Tabular Lining 对齐。

## 方案二：学术报告风格（Academic Report Style）

**设计运动**: 瑞士国际主义（Swiss International Style），强调网格系统、清晰排版和信息层次

**核心原则**:
1. 排版即设计 - 通过字体大小、粗细和间距建立视觉层次
2. 网格纪律 - 严格的12列网格系统确保对齐和秩序
3. 留白即呼吸 - 充足的留白让复杂数据更易消化
4. 色彩克制 - 仅在关键数据点使用色彩强调

**色彩哲学**: 纯白背景（#FAFAFA），主文字使用深灰（#1E293B），标题使用纯黑。强调色使用单一的深蓝（#1E40AF），辅助色使用暖灰色系。图表使用蓝色的不同明度变化（单色系）。

**布局范式**: 垂直滚动的长页面，类似学术论文的章节结构。每个章节有清晰的标题和分隔线。使用粘性导航目录（TOC）在右侧。

**签名元素**:
1. 大号衬线字体的章节编号
2. 精致的数据表格，带有交替行背景色

**交互哲学**: 滚动驱动的渐进式信息展示。图表在进入视口时才加载和动画。

**动画**: 极简动画 - 仅使用淡入和微妙的上移效果。图表使用简洁的绘制动画。

**字体系统**: 标题使用 Playfair Display（衬线体，学术感），正文使用 Source Sans 3，数据使用 Source Code Pro。

## 方案三：模块化控制台（Modular Control Console）

**设计运动**: 包豪斯功能主义（Bauhaus Functionalism）与现代SaaS仪表盘的融合

**核心原则**:
1. 模块化思维 - 每个数据维度是一个独立的可交互模块
2. 对比与张力 - 通过大小、色彩、密度的对比创造视觉兴趣
3. 直觉导航 - 信息架构符合用户的思维模型（规则→计算→结果→对比）
4. 数据叙事 - 引导用户从宏观到微观理解排产逻辑

**色彩哲学**: 浅灰暖色背景（#F8F6F3 米白），搭配深色卡片（#1C1917 暖黑）形成强烈对比。主色使用翡翠绿（#059669）代表"增长/效率"，辅以靛蓝（#4F46E5）代表"分析/洞察"，珊瑚色（#F97316）代表"注意/变化"。

**布局范式**: 非对称的卡片瀑布布局。顶部是全宽的概览区域，下方是大小不一的数据卡片。使用Tab切换不同分析维度（规则说明/期数计算/产能分析/成本对比）。

**签名元素**:
1. 圆角矩形卡片带有微妙的投影和边框渐变
2. 动态的进度环/仪表盘显示利用率

**交互哲学**: Tab驱动的多视图切换，每个视图聚焦一个分析维度。卡片内部支持展开/折叠。

**动画**: 页面切换使用弹性过渡（spring animation），数据更新使用平滑的数值变化，卡片悬停有微妙的上浮效果。

**字体系统**: 标题使用 DM Sans（几何无衬线，现代感），正文使用 Noto Sans SC，数据使用 IBM Plex Mono。
